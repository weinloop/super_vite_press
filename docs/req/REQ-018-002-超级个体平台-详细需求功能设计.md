## 一、产品范围与目标

### 1.1 产品定位

超级个体平台面向「超级个体」人群（内容创作者、独立开发者、一人公司创业者等），提供**一站式信息与成长服务**：聚合个体相关新闻与 AI 动态、提供成为超级个体的结构化教程，以及个人设置、平台介绍与帮助等通用能力。

### 1.2 终端范围

| 端 | 说明 | 首版范围 |
|----|------|----------|
| **Web 端** | 浏览器访问，响应式布局 | 全功能优先实现 |
| **APP 端** | iOS / Android 原生或混合（含 PWA） | 与 Web 功能对齐，可先采用轻量壳或 PWA |

### 1.3 目标用户（来自调研）

- 内容创作者 / KOL：需要行业与 AI 动态、教程与案例。
- 独立开发者 / 创业者：需要热点、技术趋势与实操教程。
- 交易/投资型个体：需要时效性与结构化信息。

|------|--------|------|

| **通用能力** | P1 | 个人设置、平台介绍、帮助与合规页 |

## 三、模块一：超级个体新闻

### 3.1 模块概述

为用户提供「个体新闻」与「AI 新闻」两类内容的聚合展示，数据来源于每日定时爬取的外部站点，经去重、分类后在前端以列表/流形式展示，支持按分类筛选与基础检索（可选首版）。

### 3.2 用户故事

| ID | 用户故事 | 优先级 |
|----|----------|--------|
| N-1 | 作为内容创作者，我希望在首页看到最新的个体创业/创作相关新闻，以便快速获取选题与趋势。 | P0 |
| N-2 | 作为独立开发者，我希望看到与 AI 产品、技术、政策相关的最新新闻，以便把握技术方向。 | P0 |
| N-3 | 作为用户，我希望能区分「个体新闻」和「AI 新闻」两类内容，并可按分类筛选查看。 | P0 |
| N-4 | 作为用户，我希望每条新闻包含标题、摘要、来源、发布时间，并可跳转到原文链接。 | P0 |
| N-5 | 作为用户，我希望新闻列表支持下拉刷新/分页加载，以便持续浏览。 | P0 |

### 3.3 功能需求详述

#### 3.3.1 新闻分类与展示

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 分类定义 | 固定两类：**个体新闻**、**AI 新闻**。支持在首页通过 Tab/筛选项切换。 | 前端可切换两类列表；两类数据来源与展示逻辑分离。 |
| 列表展示 | 列表项展示：标题、摘要（可选截断）、来源名称、发布时间；点击进入详情或跳转原文。 | 标题与时间必显；来源与摘要符合设计稿；点击行为一致。 |
| 详情/跳转 | 详情页可仅展示摘要+原文链接按钮，或内嵌原文链接；须明确版权与来源。 | 用户可一键跳转至原文；展示「来源：xxx」及合规说明。 |

#### 3.3.2 爬取与数据管理

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 爬取策略 | 每日定时任务从预设数据源爬取；遵守 robots.txt 与站点条款；优先使用官方 API（若有）。 | 有明确任务调度（如每日 1 次）；有数据源白名单与合规说明文档。 |
| 数据存储 | 爬取结果持久化存储；支持按分类、时间查询；支持去重（如基于 URL 或唯一标识）。 | 同一 URL 不重复展示；支持按日/分类查询。 |
| 数据源管理 | 数据源可配置（来源名称、URL 规则、分类归属、是否启用）；个体新闻与 AI 新闻对应不同源或标签。 | 后台或配置可维护数据源列表；新源可归属「个体」或「AI」。 |

#### 3.3.3 首版可选增强

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 关键词搜索 | 在新闻标题/摘要中搜索。 | P2 |
| 收藏/稍后读 | 用户登录后收藏单条新闻。 | P2 |
| AI 摘要 | 对爬取正文做 AI 摘要后展示。 | P2 |

### 3.4 页面与信息结构（功能设计）

- **页面**：新闻首页（可选：新闻详情页或直接外链）。
- **信息结构**：
  - 顶部：分类 Tab「个体新闻 | AI 新闻」。
  - 列表：卡片或列表项（标题、摘要、来源、时间、原文链接）。
  - 底部：分页或「加载更多」。
- **交互**：切换 Tab 刷新列表；下拉刷新（APP/Web 均可）；点击条目跳转详情或原文。

### 3.5 非功能约束

- **合规**：仅爬取允许的站点；展示时保留来源与原文链接；必要时仅存标题+摘要+链接。
- **性能**：列表分页或懒加载，单页条数上限可配置（如 20 条）。

## 四、模块二：如何成为超级个体（教程）

### 4.1 模块概述

提供结构化教程/指南，帮助用户理解并实践「成为超级个体」的路径，包括阶段划分、主题分类、精选内容聚合，可与新闻/案例联动（如「本周案例」）。

### 4.2 用户故事

| ID | 用户故事 | 优先级 |
|----|----------|--------|
| T-1 | 作为新手，我希望看到清晰的「从零到超级个体」的阶段划分，以便知道下一步该做什么。 | P1 |
| T-2 | 作为用户，我希望能按主题（如工具、AI 协作、内容创作、一人公司）浏览教程。 | P1 |
| T-3 | 作为用户，我希望教程内容包含标题、摘要、正文或外链，并可与新闻/案例关联。 | P1 |

### 4.3 功能需求详述

#### 4.3.1 教程结构

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 阶段结构 | 支持多阶段（如入门→进阶→变现），每阶段下可挂载多篇教程或链接。 | 前端展示阶段导航；阶段与教程关联关系可配置。 |
| 主题分类 | 支持多主题（如工具、AI 协作、内容创作、一人公司、政策与合规等）；一篇教程可属多主题。 | 支持按主题筛选/导航；主题列表可配置。 |
| 教程条目 | 每条教程包含：标题、简短摘要、正文或外链、可选配图、可选「关联新闻/案例」链接。 | 列表展示标题+摘要；详情展示完整内容或跳转外链。 |

#### 4.3.2 内容来源与维护

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 内容来源 | 支持自产、精选转载（带来源）、外链聚合；外链需标明来源。 | 每条内容有明确来源标识；转载/外链符合版权约定。 |
| 与新闻联动 | 可选在教程详情或专区展示「相关新闻」「本周案例」等，数据来自新闻模块。 | 可配置展示规则（如按标签、关键词）；无则隐藏该区块。 |

### 4.4 页面与信息结构（功能设计）

- **页面**：教程首页（阶段/主题导航 + 教程列表）、教程详情页（正文或外链跳转）。
- **信息结构**：
  - 教程首页：阶段 Tab 或步骤条 + 主题筛选项 + 教程卡片列表（标题、摘要、主题标签）。
  - 教程详情：标题、摘要、正文/外链按钮、可选「相关新闻」「本周案例」区块。
- **交互**：点击阶段/主题筛选列表；点击教程进入详情；外链在新窗口打开。

### 4.5 首版范围建议

- 先上线阶段与主题框架 + 少量精品教程（5～10 篇），后续再扩展内容与「本周案例」联动。

## 五、模块三：通用能力

### 5.1 模块概述

包含个人设置、平台介绍、帮助与说明、以及合规相关页面，保证基础体验与合规。

### 5.2 个人设置

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 设置入口 | Web 与 APP 均提供「设置」或「个人中心」入口。 | 入口位置符合常见产品习惯。 |
| 通知设置 | 若支持消息推送，需提供通知开关（总开关及分类）。 | 用户可关闭/开启推送。 |
| 主题与显示 | 可选支持浅色/深色主题、字体大小等。 | 首版可仅支持主题切换。 |
| 账号与安全 | 若支持登录，需提供账号信息展示、修改与登出。 | 登出后清除本地登录态。 |

### 5.3 平台介绍与帮助

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 平台介绍 | 「关于我们」或「平台介绍」页：产品定位、核心功能、联系方式（可选）。 | 内容与产品定位一致，可配置更新。 |
| 帮助/FAQ | 常见问题与使用说明，支持列表+展开或独立帮助页。 | 至少覆盖：如何看新闻、如何用教程、如何反馈。 |
| 反馈入口 | 用户可提交意见或 Bug 反馈（表单或邮件链接）。 | 提交后有明显成功提示或邮件确认。 |

### 5.4 合规页面

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 用户协议 | 注册/使用前可查看用户协议。 | 在注册流程或设置中可随时查看。 |
| 隐私政策 | 说明数据收集、使用、存储与第三方共享。 | 内容完整，在注册或首次使用可查看并可从设置进入。 |

### 5.5 页面与信息结构（功能设计）

- **页面**：设置页（含子项：通知、主题、账号、关于、帮助、隐私与协议）、关于我们、帮助/FAQ、用户协议、隐私政策。
- **信息结构**：设置采用列表式菜单，每项进入对应子页或弹层；关于/帮助/协议为静态内容页，支持从设置或页脚链接进入。

## 七、非功能需求（通用）

| 类型 | 描述 |
|------|------|
| **多端一致性** | Web 与 APP 核心功能与信息架构一致；视觉与交互可依端做适配。 |
| **性能** | 列表首屏加载时间可接受（建议 &lt; 3s）；爬取任务不阻塞前台访问。 |
| **合规** | 爬虫遵守 robots.txt 与数据源条款；用户协议与隐私政策在首版上线前就绪。 |
| **可维护性** | 数据源、阶段/主题、教程内容、静态页文案均支持配置或后台维护，避免硬编码。 |

## 八、后续可拆分的实现级需求

本文档可作为后续「设计说明书」与「开发任务」的输入。实现时建议拆分为：

- 新闻模块：数据源配置与爬虫、存储与 API、Web/APP 新闻列表与详情。
- 教程模块：教程 CMS 或配置结构、教程首页与详情页、与新闻的联动接口。
- 通用模块：设置页、关于/帮助/协议静态页、合规文案终稿。
